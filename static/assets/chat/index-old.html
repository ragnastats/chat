Welcome to RagnaStats!

RagnaStats is a search engine for the MMORPG Ragnarok Online. The information in our databases is a collaborative effort, constantly updated by players and automated programs. We collect information from all iRO servers to help you find the best deals on items, monster locations, and quest NPCs.

Developer Notes

  • The new ragnastats database needs to store all the information presented in the navigation on this page.
  • It also needs to be able to verify information from multiple sources.
  • However, some information may change frequently, so different thresholds should apply for shops, player information, etc.
  • Maybe users should be weighted based on how much good data they've already sent?
  • That probably won't be implimented in the first version, but try to keep things open ended enough to possibly support it.
  • We don't want to display shops with inaccurate information, but what equipment someone is wearing should be more forgiving. (Only need verification from one or two sources)
  • There could also be a time element for non-shop information. This would give a chance for other users to see the same thing, or at the very least prevent someone from submitting false data and seeing it appear on the site immediately. (Script kiddies get bored)
  • We also need to store information about sprites. Every sprite. Check out the sprites alpha page.
  • It'll be important to remember the original directories/file names, but we also need to present the information with SEO friendly URLs, easy navigation, etc.
  • What needs to be changed in the current database to better work with the new system?
  • After messing with the original RagnaStats source all day, I found some unindexed queries which were the source of the site's slow loading times. Even with almost 1,000,000 shops in the database it's once again loading super fast.
  • But for portability reasons (if we ever need to move servers?) it would still be convenient to have a table which only contains market information from the past month.
  • What operations are CPU/load time intensive? What should be cached? Daily item averages?
